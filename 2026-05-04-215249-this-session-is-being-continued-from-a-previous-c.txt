  Your four concerns

  1) "Raydium pool has admin as not me but vault, why?"

  Two clarifications:
  - The Raydium CPMM pool itself has no admin. It's a permissionless AMM —
  anyone can deposit LP, anyone with LP tokens can withdraw. There is no
  on-chain admin field.
  - The pool LP tokens are owned by your deploy wallet (id.json), not the vault.
   scripts/create_raydium_pool.ts:88-89 uses ANCHOR_WALLET=$DEPLOY_KEYPAIR (your
   id.json). The vault PDA never touches LP.

  What you're seeing in the frontend is the deposit-form admin gate
  (RaydiumLiquidityForm.tsx:25):
  const isAdmin = Boolean(publicKey && publicKey.equals(VAULT_ADMIN));
  VAULT_ADMIN = NEXT_PUBLIC_VAULT_ADMIN = the spectraq_admin.json keypair that
  demo.sh auto-generates as a separate identity from your deploy wallet
  (demo.sh:99-103, intentional separation: program upgrade authority ≠ vault
  admin). When you connect your normal Phantom wallet (which is the deploy
  keypair), the form correctly says "not admin" because the pubkeys don't match.
   Two options:

  - a) Import ~/.config/solana/spectraq_admin.json into Phantom and connect that
   — the form unlocks.
  - b) Collapse the two identities by setting
  SPECTRAQ_ADMIN_KEYPAIR="$HOME/.config/solana/id.json" in .env and re-running
  demo.sh. Loses the security separation but matches what you expect.

  2) "Agent trades shown as failed — is end-to-end logic correct?"

  No, there's a real bug. This is why every sell-side trade fails:

  The on-chain program requires last_signal == -1 for SolToUsdc
  (execute_trade.rs:103):
  TradeDirection::SolToUsdc => {
      require!(vault.last_signal == -1, VaultError::SignalDirectionMismatch);
  }

  But mockComputeSignal (arcium.ts:104) only ever emits 0 or 1 (binary):
  return left > right ? 1 : 0;

  And decideTrade calls solToUsdc with signal === 0 (trader.ts:111-119).

  End-to-end: agent says "signal=0, sell" → stamps last_signal=0 on-chain →
  tries SolToUsdc → program checks last_signal == -1 → fails with
  SignalDirectionMismatch (6008). Every time. That's exactly what your log shows
   from 19:21–19:25.

  The first session-1 swap that actually landed worked because it was a BUY
  (signal=1, UsdcToSol, last_signal == 1 matches). The mismatch only shows up on
   sells.

  Fix: make mock mode emit -1 for "go to USDC" instead of 0, and align
  decideTrade to the tristate {-1, 0, 1}. The program is already correct; the
  mock signal generator is wrong.

  3) "Agent does not take trades, like almost no trades"

  Two compounding causes from the most recent log:

  - Position classifier returns split. Current vault is ~$28 USDC + 0.31 SOL ≈
  28M:31M e6 — within 10× of each other (trader.ts:159-163), so classifier says
  split. With signal=1 + split, decideTrade returns null (trader.ts:123) — by
  design, "split is treated as USDC-leaning so the next BUY is suppressed."
  - Even when signal flipped to 0, sells failed (issue #2 above).

  So the agent is wedged: never buys (split + 1 → no-churn), never sells
  (signal=0 + program requires -1 → on-chain reject). 0 trades on the last run
  is consistent with this.

  4) "Verify the vault NAV is/isn't artificially shown"

  Vault NAV is real, not artificial. frontend/app/api/vault/route.ts:120-121:
  const navUsdcE6 = priceE6 != null
    ? usdcBalance + (solBalance * priceE6) / 1_000_000_000n
    : usdcBalance;
  - usdcBalance / solBalance come from the on-chain vault_state Anchor account
  (cached, refreshed at every settle).
  - priceE6 is the live Pyth SOL/USD oracle price (readPythPriceE6,
  vault/route.ts:96).

  The agent uses the same formula on live ATA balances + live Pyth
  (agent/src/index.ts:212-213). Both sides agree. Only nav24hChange is null
  (placeholder — explicitly noted as "no real history" at line 124), but the
  displayed NAV number itself is correct.

  doesn't start in split).
  3. Patch demo.sh: drop the broken Helius URL, move/extend the
  frontend/.env.local writer, propagate RPC_URL into the agent, add /app/pool to
   the summary.
  4. Decide on the admin model (separate spectraq_admin vs collapse onto
  id.json).
