● 1. How the signal is decided

  The agent uses a classic moving-average (MA) crossover strategy on SOL price.

  Inputs (refreshed every tick = 60s):
  - 50 most recent 1-minute closes of SOL/USDT pulled from Binance REST

  STRATEGY_SLOW_N=30        # long window
  STRATEGY_THRESHOLD_BPS=0  # confirmation buffer

  Computation (agent/src/arcium.ts:83-108):
  fast_avg = mean of last 10 closes
  slow_avg = mean of last 30 closes

  signal = +1   if fast_avg  >  slow_avg × (1 + threshold/10000)
  signal = -1   otherwise

  Mapping to trades (agent/src/trader.ts:113-132):

  ┌────────┬───────────────────────────────────┬──────────────┬────────────┐
  │ signal │              meaning              │  direction   │    size    │
  ├────────┼───────────────────────────────────┼──────────────┼────────────┤

  │        │ trailing 30m → uptrend            │ (buy)        │ vault USDC │
  ├────────┼───────────────────────────────────┼──────────────┼────────────┤

  │        │ trailing 30m → downtrend / flat   │ (sell)       │ vault SOL  │
  ├────────┼───────────────────────────────────┼──────────────┼────────────┤

  └────────┴───────────────────────────────────┴──────────────┴────────────┘

  First depositor:  shares_minted = usdc_deposited                 (1:1)
  Later depositor:  shares_minted = (usdc_deposited × total_shares) / nav
  Withdraw:         usdc_out = (shares_burned × nav_usdc_part) / total_shares

  nav (net asset value) = vault's USDC balance + (vault's SOL × Pyth price).
  Recomputed live on every deposit/withdraw — you can never deposit at stale
  prices.

  - Agent trades. If SOL pumps and the vault is long SOL, NAV rises.

  - You redeem 100 shares → you get back $120 of value (USDC + SOL mixed, you
  can just sell the SOL for USDC yourself if you want all dollars).
  - Conversely, if the agent loses, nav_per_share falls and you get less back.
